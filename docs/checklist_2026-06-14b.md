## 2. 회원가입 — 이미 가입된 이메일 처리 개선

배경:
- `handleSignUpOnly()`/`handleJoinSubmit()` 실패 시 wizard choice 팝업이 그대로 남아
  "다시 팝업이 나온다"고 느끼는 UX 문제.
- 422 오류(이미 가입된 이메일)가 발생해도 wizard가 닫히지 않아 사용자가 혼란을 겪음.
수정: "이미 가입된 이메일" 오류 시 wizard 닫고 로그인 탭으로 자동 전환 + 이메일 자동 입력.

### 정상 가입 경로 (회귀 확인)

- [ ] 새 이메일로 회원가입 → 이름/비밀번호/확인 → choice 선택
  - "기존 조직에 가입하기" 선택 → 처리 완료 → 로그인 후 PendingPage 표시
  - "내 서비스 시작하기" 선택 → 서비스 이름/전화번호 → 완료 후 이동

### 이미 가입된 이메일 시나리오

- [ ] 이미 가입된 이메일로 회원가입 시도
  - `handleEmailNext()` 단계: "이미 가입된 이메일입니다. 로그인을 이용해 주세요." 표시 (기존 동작)
- [ ] auth.users에는 있으나 profiles에 없는 엣지 케이스:
  - 이메일 입력 → wizard 열림 → name/password/confirm → choice → 기존 조직 가입 선택
  - `signUp()` 422 반환 → **wizard 닫힘** + **로그인 탭으로 전환** + 이메일 자동 입력 + 에러 메시지 표시
- [ ] 내 서비스 시작하기 경로에서 422 발생 시 동일하게 로그인 탭으로 전환
