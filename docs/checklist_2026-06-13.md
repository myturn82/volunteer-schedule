## 1. 스케줄 초기화 복구 기능

배경: 초기화 후 되돌릴 방법이 없어, 실수로 삭제 시 복구 수단 필요.
초기화 전 스냅샷을 DB(`assignment_snapshots`)에 저장하고 복구 배너로 복원.

### DB 사전 적용 확인

- [ ] **[DB 확인]** Supabase SQL Editor에서 `049_assignment_snapshots.sql` 실행 완료
- [ ] **[DB 확인]** `assignment_snapshots` 테이블 생성 확인 (`SELECT * FROM assignment_snapshots LIMIT 1;`)
- [ ] **[DB 확인]** RLS 정책 `snapshots_admin_all` 적용 확인

### 월간 뷰 — 초기화 후 복구

- [ ] 월간 뷰에서 배정 있는 달로 이동
- [ ] 메뉴 → 초기화 → 확인 다이얼로그에 "초기화 후 복구할 수 있습니다." 문구 표시
- [ ] 삭제 실행 후 하단에 `N건 초기화됨 | 복구하기 | ✕` 배너 표시
- [ ] **[DB 확인]** `SELECT * FROM assignment_snapshots;` — 스냅샷 행 1건 생성
- [ ] 복구하기 클릭 → "초기화 이후 등록된 스케줄은 덮어써집니다" 경고 표시
- [ ] 복구 확인 → 스케줄 복원됨 (배정 건수 일치)
- [ ] **[DB 확인]** 복구 완료 후 `assignment_snapshots` 행 자동 삭제
- [ ] 복구 후 `N건 스케줄이 복구됐습니다.` 토스트 표시

### 복구 배너 닫기(✕)

- [ ] ✕ 버튼 클릭 시 배너만 사라지고 스냅샷은 DB에 남아 있음
- [ ] **[DB 확인]** `SELECT * FROM assignment_snapshots;` — 행 존재

### 재초기화 시 스냅샷 교체

- [ ] 초기화 → 복구하지 않고 다시 초기화 → 이전 스냅샷이 새 스냅샷으로 교체됨
- [ ] **[DB 확인]** `assignment_snapshots`에 같은 tenant+year+month 행이 1건만 존재

### 주간 뷰 — 초기화 후 복구

- [ ] 주간 뷰에서 초기화 실행 → 복구 배너 표시
- [ ] 복구 확인 → 해당 주의 스케줄 복원됨
- [ ] 주 뷰가 월 경계를 넘는 경우(예: 6/29~7/5)에도 양쪽 월 스케줄 복원됨

### 일간 뷰 — 초기화 후 복구

- [ ] 일간 뷰에서 특정 날짜 초기화 → 복구 배너 표시
- [ ] 복구 확인 → 해당 날짜 스케줄 복원됨

### 잠금 배정 처리

- [ ] 잠금 배정이 있는 달 초기화 시 잠금 건은 초기화되지 않고 유지
- [ ] 복구 시 스냅샷에 잠금 배정이 없으므로 잠금 건에 영향 없음
