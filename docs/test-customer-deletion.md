# 고객 삭제 기능 테스트 체크리스트

테스트 환경: http://localhost:5173 → 슈퍼관리자 페이지

---

## 1. 고객 직접 삭제 (영구 삭제)

- [ ] 고객 행의 **삭제** 버튼 클릭 → 확인 모달 열림
- [ ] 모달에 고객명, 소속 조직 수 표시됨
- [ ] 소속 조직이 있을 때 → 주황색 경고 문구 표시됨 (`소속 조직 N개와 모든 데이터가 영구 삭제됩니다`)
- [ ] 고객명 입력 전 → **영구 삭제** 버튼 비활성화
- [ ] 고객명 틀리게 입력 → 버튼 여전히 비활성화
- [ ] 고객명 정확히 입력 → 버튼 활성화
- [ ] 삭제 실행 후 → 고객 목록에서 해당 고객 사라짐
- [ ] **[DB 확인]** 연결된 tenant도 함께 삭제됨 (CASCADE)
- [ ] **[DB 확인]** 삭제된 tenant 소속 회원의 조직 선택 화면에서 해당 조직 사라짐

---

## 2. 고객 비활성화 / 복구

- [ ] **비활성화** 버튼 클릭 → `is_active = false`로 변경
- [ ] 비활성화된 고객 행 → 버튼이 **복구**로 바뀜
- [ ] **[DB 확인]** 연결된 tenant의 `is_active`도 `false`로 변경됨 (트리거)
- [ ] 비활성화된 tenant의 멤버 → 로그인 후 해당 조직 목록에 안 보임
- [ ] **복구** 버튼 클릭 → `is_active = true`, 버튼 다시 **비활성화**로 바뀜
- [ ] **[DB 확인]** 복구 시 연결된 tenant의 `is_active`도 `true`로 복원됨

---

## 3. 탈퇴 요청 섹션 (`deletion_requested_at`)

- [ ] `deletion_requested_at`이 있는 고객 → 상단 **탈퇴 요청 중** 섹션에 표시됨
- [ ] 경과 일수 올바르게 표시됨
- [ ] 30일 미만 → **완전 삭제 ⚠** (경계 스타일, 빨간 테두리)
- [ ] 30일 이상 → **완전 삭제** (빨간 배경) + **삭제 가능** 배지 표시
- [ ] **복구** 클릭 → `is_active = true`, `deletion_requested_at = null`, 목록에서 제거
- [ ] **완전 삭제** 클릭 → 확인 모달 → 삭제 실행 후 탈퇴 요청 목록 + 고객 목록 양쪽에서 제거
- [ ] **[DB 확인]** 완전 삭제 시 연결된 tenant도 CASCADE 삭제됨

---

## 4. 시스템 고객 보호

- [ ] `id = 00000000-0000-0000-0000-000000000001` (시스템) → 비활성화·삭제 버튼 없음

---

## 테스트 우선순위

1. **1번** 영구 삭제 — CASCADE 동작 핵심 검증
2. **2번** 비활성화/복구 — 트리거 동작 검증
3. **3번** 탈퇴 요청 — 30일 분기 로직 검증
